## 1. 项目目标

将一个基于 React + sql.js 的 SQLite 可视化管理工具，移植为 VSCode 扩展。用户在 VSCode 文件资源管理器中**点击 `.db` / `.sqlite` / `.sqlite3` 文件即可打开可视化预览**，进行表数据浏览与增删改查、执行 SQL 语句。

## 2. 范围

### 2.1 包含（核心功能）

- **自定义编辑器**：点击 SQLite 文件 → 打开可视化管理界面（Webview）。
- **表管理**
  - 侧边栏表列表（含每张表的行数）。
  - 多 Tab 切换（每张表一个 Tab）。
  - 表数据浏览：表格展示、分页、排序。
  - 数据搜索：全表关键字搜索 / 指定字段搜索。
  - 数据操作：新增行、行内编辑、删除行（二次确认）。
  - NULL 值处理、主键字段保护。
  - 查看建表 SQL、重命名表。
  - 新建表面板（表单 → 建表 SQL 预览 → 执行）。
- **SQL 执行器**：基于 Monaco 的 SQL 编辑器，执行查询/写操作，展示结果，保留执行历史。
- **主题**：默认跟随 VSCode 明暗主题，支持手动切换。
- **保存模型**：采用 VSCode 原生脏标记 + `Ctrl/Cmd+S` 保存，支持撤销（revert）与热退出备份。

### 2.2 不包含（本期丢弃）

- **AI 助手**：LangChain 工具调用聊天（`store/ai`、`store/conversation`、`DatabaseInfoPanel` 等）。
- **数据库列表管理页**（`DatabaseManagement`）：导入/新建/删除数据库文件 —— 该能力由 VSCode 文件资源管理器天然替代，每个编辑器只管打开的那一个文件。
- **参考项目中硬编码的 AI API Key**：不会迁移到新项目。
- uTools 专属逻辑（`preload.js`、`utools/`、`mockApis`、`window.utools`）。

### 2.3 改写

| 原实现 | 新实现 |
|--------|--------|
| `window.customApis`（uTools preload 提供的同步桥接） | Webview ↔ 扩展宿主的 postMessage 协议 |
| `databaseStore` 多数据库列表 + 当前库 ID | 简化为"当前打开的单个文件" |
| sql.js 写回文件直接落盘 | 经宿主走 VSCode 脏标记/保存流程 |
| `localStorage` mock 文件系统 | 不需要；宿主用 Node `fs` |
| 设置存储（uTools dbStorage） | VSCode `globalState` / `workspace` 配置 |

## 4. 里程碑与分阶段任务

### M0 — 脚手架（可加载空插件）
- 初始化扩展 `package.json`（贡献点：customEditors）。
- esbuild 构建扩展宿主；Vite 构建 Webview。
- tsconfig、目录结构、最小 `extension.ts` + 空 Webview。
- 验收：F5 启动 Extension Development Host，点击 `.db` 文件能打开一个空面板。

### M1 — 桥接打通 + 表浏览（最小可用）
- 实现 `CustomEditorProvider`：读取文件字节、推送给 Webview。
- 实现 postMessage 桥接层（替代 customApis 的文件读取）。
- 适配 `SQLiteHelper`，sql.js 在 Webview 内存加载数据库。
- 移植侧边栏表列表 + 表数据面板（只读浏览 + 分页 + 排序）。
- 验收：打开真实 `.db` 文件，能看到表列表、浏览分页数据。

### M2 — 增删改查 + 保存
- 移植新增/编辑/删除行、搜索、NULL 处理、主键保护。
- 接通保存模型：sql.js 导出字节 → 宿主标脏 → `Ctrl+S` 写回磁盘 → revert/备份。
- 验收：改数据后保存，重开文件数据已变；未保存时编辑器显示脏标记。

### M3 — 新建表 + SQL 执行器
- 移植新建表面板（FormToTable + SQL 预览 + 执行）、重命名表、查看建表 SQL。
- 移植 Monaco SQL 执行器（执行、结果展示、历史）。
- 验收：能新建表、执行任意 SQL、查看结果。

### M4 — 主题与打磨
- ThemeProvider 跟随 VSCode 明暗，支持手动切换。
- 样式适配、边界与错误处理、空状态、加载态。
- 验收：明暗主题切换正常，交互流畅。

## 5. 风险点与对策

| 风险 | 对策 |
|------|------|
| sql.js wasm 在 Webview 的资源路径加载 | 通过 `webview.asWebviewUri` 注入 wasm 路径；`locateFile` 改为读取注入的 URI |
| Monaco 静态资源在 Webview/CSP 下加载 | 将 Monaco min 资源随 Webview 打包，配置 CSP 与 worker 路径 |
| 原 `customApis` 为**同步** API，postMessage 为**异步** | SQLiteHelper 改造为异步初始化（init/save/reload 已是 async，读文件改异步获取） |
| 大文件读入内存（sql.js 全量加载） | M1 先支持常规体量；超大文件给出提示，后续可优化 |
| Webview CSP 限制内联脚本/样式 | 使用 nonce、`asWebviewUri`，Ant Design CSS-in-JS 经 StyleProvider 注入 |
| 二进制数据在 host↔webview 传输 | 以 `Uint8Array`/`ArrayBuffer` 传输，避免 base64 膨胀（必要时退回 base64） |
| 保存与外部修改冲突 | 借助 VSCode CustomDocument 的 backup/revert 机制处理 |

## 6. 验收总标准

- 点击 `.sqlite`/`.db`/`.sqlite3` 文件可打开可视化界面。
- 表浏览、分页、排序、搜索、增删改查可用。
- 修改可保存回磁盘，支持脏标记与撤销。
- 可新建表、执行 SQL、查看结果与历史。
- 明暗主题正常，界面风格沿袭参考项目。
