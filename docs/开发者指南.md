# sqlite-tool 开发者指南

本文档面向参与 sqlite-tool 开发、调试、验证和打包发布的开发者。用户安装和使用说明请查看根目录 [README.md](../README.md)。

## 项目结构

```txt
扩展宿主 (Node)  ── postMessage ──  Webview (Chromium 沙箱)
SqliteEditorProvider                React 19 + Ant Design 6
SqliteDocument(字节模型)            sql.js(WASM 内存库) + Monaco
文件 IO / 脏标记 / 保存 / 备份       bridge.ts(异步桥) + SQLiteHelper
```

- `src/`：VS Code 扩展宿主代码，负责注册自定义编辑器、读写数据库文件、脏标记、保存、撤销、备份和 Webview 消息桥接。
- `webview/`：React Webview 应用，负责 SQLite 可视化界面、sql.js 内存数据库操作、SQL 编辑器和用户设置。
- `scripts/`：样本数据库生成与核心逻辑自检脚本。
- `package.nls*.json`：扩展清单本地化文案。
- `README*.md`：面向用户的多语言说明。

## 开发环境

```bash
npm install
npm run build
```

常用脚本：

```bash
npm run dev:webview  # 启动 Webview Vite HMR 开发服务器
npm run watch:ext    # 监听并构建扩展宿主
npm run watch:webview # 监听构建 Webview 产物，不启用 HMR
npm run build        # 构建 Webview 与扩展宿主
```

## VS Code 调试

在 VS Code 中按 `F5` 启动 Extension Development Host。

- `运行扩展（热更新）`：自动启动 Vite HMR 与宿主 watch。修改 `webview/src/**` 后，当前打开的数据库页面会热更新，无需重新 build。
- `运行扩展`：使用构建产物模式，适合验证打包前状态。

热更新模式通过环境变量把扩展宿主指向 Vite 服务：

```bash
SQLITE_TOOL_WEBVIEW_DEV_SERVER=http://localhost:19753
```

说明：

- `webview` 前端代码支持热更新。
- `src/**` 扩展宿主代码变更后会自动重新编译到 `dist/`，但仍需在 Extension Development Host 中执行一次“开发人员: 重新加载窗口”或重新 F5。VS Code 扩展宿主本身不支持真正意义上的无感热替换。

## 样本数据库

生成测试数据库：

```bash
npm run sample:db
```

生成结果默认输出到 `db/`，该目录已加入 `.gitignore`。样本包含：

- `sample-mixed.db`：常规混合场景，含多表、超长列名、超长文本、NULL、带空格列名。
- `sample-wide.sqlite`：宽表场景，25 列，覆盖横向滚动与长表头展示。
- `sample-large.sqlite3`：大数据量场景，`event_logs` 表 5200 行，含长文本与 JSON 文本列。

## 自检脚本

逻辑自检脚本无需启动 VS Code：

```bash
node scripts/verify-mutations.cjs  # 增删改 + 字节回写
node scripts/verify-m3.cjs         # 建表 / 重命名 / SQL 分类
```

建议在涉及 SQLiteHelper、保存模型、SQL 执行或表结构变更时运行这些脚本。

## 保存模型

- 宿主是唯一直接访问磁盘的一方。
- Webview 使用 sql.js 在内存中加载和操作数据库。
- 数据变更后，Webview 导出数据库字节并通过 postMessage 发送给宿主。
- 宿主更新 `SqliteDocument` 字节快照，并触发 VS Code 自定义编辑器脏标记。
- 用户按 `Ctrl/Cmd+S` 后，宿主把当前字节写回磁盘。
- 如果磁盘文件在外部发生变化，保存前会提示冲突风险。

## 打包

```bash
npm run package
```

该命令会先执行 `npm run build`，再通过 `vsce package` 生成 `.vsix`。

本地安装打包结果：

```bash
code --install-extension sqlite-tool-0.0.1.vsix
```

## 发布准备

正式发布前至少检查：

- `package.json` 中的 `name`、`displayName`、`description`、`publisher`、`version` 是否正确。
- README 是否面向最终用户，且语言版本链接可访问。
- `npm run build` 是否通过。
- 必要的自检脚本是否通过。
- 生成的 `.vsix` 是否能在 Extension Development Host 或本地 VS Code 中正常安装和打开数据库。

发布到 VS Code Marketplace 时使用：

```bash
vsce publish
```

当前 `publisher` 仍为 `local`，正式发布前需要替换为 Marketplace 中注册的 publisher id。
